# Security Policy

This repository follows a responsible disclosure policy. If you discover a security vulnerability, please report it privately so maintainers can triage and remediate before public disclosure.

Reporting
- Preferred: create a private GitHub Security Advisory for the repository (recommended).
- Alternative: contact the maintainers by the private contact listed in the repository metadata (see `LICENSE` and crate manifests).

When reporting, please include:
- Affected version(s) or commit SHA
- Clear steps to reproduce the issue
- Proof-of-concept or exploit details, if available
- Impact assessment (confidential if necessary)

Triage and Response
- Maintainers will acknowledge receipt within 72 hours and provide a planned timeline for fixes.
- For critical vulnerabilities, maintainers will coordinate an immediate private disclosure and a patch release.

Do not create a public issue for active security vulnerabilities — use a private advisory or the maintainers' contact method.

PGP/Encryption
- If you need to share sensitive exploit details encrypted, contact the maintainers for the preferred key or channel.

Thank you for helping keep this project secure.
